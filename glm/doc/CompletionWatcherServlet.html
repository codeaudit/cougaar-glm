Introduction

The GLM Completion Watcher Servlet allows the user to measure how long it takes for an
agent to complete its tasks.  In other words, it measures the time it takes for an agent to move
from an “idle” agent state to a “done” agent state.  In terms of tasks, this is from the input of the
first task to the completion of the last.  It's a close cousin of the Completion Servlet.

But what does it mean for a task to be complete?

A task is complete when
  • A task has a plan element
  • The plan element has an Allocation Result (Estimated or Reported)
  • The Allocation Result’s confidence reaches 100%
This definition requires plugins to play by the rules and attach plan elements, propagate allocation results, etc.

Servlet Parameters

The servlet has a few parameters :
  • the first quiet interval
    • parameter : firstInterval (defaults to 10 seconds)
    • units : seconds
  • the second quiet interval
    • parameter : secondInterval (defaults to 10 seconds)
    • units : seconds
  • show results as either html, xml, or serialized java objects
    • parameter : format = html, xml, or data

Returned Result Format

The execution time returned can be in either html, xml, or serialized
data format.

Timing Information

You get back timing information like this :

<response>
 <results totalTime="">
  <task id="interval" time="0:35:501"/>
 </results>
 <histogram minBucket="13" maxBucket="13">
  <bucket elapsedTime="20000..50000" count="1"/>
 </histogram>
</response>

This is the html that the GLMStimulatorServletResponseData produces.

Sample URL

A sample URL access of the servlet is

http://localhost:8800/$TRANSCOM/watcher?firstInterval=5&secondInterval=5&format=html

Necessary Plugins

The necessary component is (as from an ini file) :

plugin = org.cougaar.glm.servlet.CompletionWatcherServletComponent(org.cougaar.glm.servlet.CompletionWatcherServlet, /watcher)
